# AGENTS.md

## 1. 문서 목적

이 문서는 Antigravity AI 코딩 에이전트가 `감성 분석 서비스`를 구현할 때 반드시 지켜야 할 작업 규칙이다.  
이 프로젝트는 비전공자 학습자가 이해할 수 있는 단순한 구조를 유지하면서, 실제 배포 가능한 MVP를 만드는 것을 목표로 한다.

---

## 2. 프로젝트 한 줄 정의

사용자가 입력한 텍스트를 OpenAI API로 감성 분석하고, 결과를 `긍정 / 부정 / 중립` 라벨과 결과별 색상, `신뢰도`, `분석 이유`, `오류 메시지`로 보여주는 웹 서비스이다.

서비스 콘셉트명은 다음으로 통일한다.

```text
Sentiment Forest / 감정의 숲
```

## 4. 구현 범위 고정

### 4.1 반드시 구현할 것

- 텍스트 입력 화면
- 분석 요청 버튼
- 로딩 상태 표시
- OpenAI API를 호출하는 Node.js 서버 API
- 감성 결과 표시
  - 긍정: 초록색 계열 배지
  - 부정: 붉은색 계열 배지
  - 중립: 회색 계열 배지
- 색상만으로 결과를 구분하지 않고, 반드시 텍스트 라벨도 함께 표시
- 신뢰도 백분율 표시
- 분석 이유 표시
- 사용자 입력 오류 메시지 표시
- 서버 / OpenAI / Supabase 오류 메시지 표시
- Supabase에 분석 로그 저장
- Vercel 배포 가능한 구조

### 4.2 구현하지 말 것

아래 기능은 MVP 범위가 아니다. 임의로 추가하지 않는다.

- 로그인 / 회원가입
- 관리자 페이지
- 결제 기능
- 댓글 기능
- 다국어 전환 UI
- 분석 기록 전체 조회 화면
- 파일 업로드 분석
- 음성 분석
- 이미지 분석
- 실시간 스트리밍 응답
- 복잡한 차트 대시보드
- 프레임워크 기반 SPA로 전환
  - React, Vue, Next.js 등은 사용하지 않는다.

## 6. 보안 규칙

### 6.1 절대 금지

아래 값은 절대 프론트엔드 코드에 넣지 않는다.

```text
OPENAI_API_KEY
SUPABASE_SERVICE_ROLE_KEY
```

금지 예시:

```js
// 금지
const apiKey = "sk-...";
```

### 6.2 환경 변수만 사용

서버 API는 반드시 다음 환경 변수를 사용한다.

```text
OPENAI_API_KEY
OPENAI_MODEL
SUPABASE_URL
SUPABASE_SERVICE_ROLE_KEY
```

### 6.3 사용자 입력 처리

- 사용자의 입력값은 반드시 `trim()` 처리한다.
- 빈 문자열은 서버 호출 전에 프론트엔드에서 1차 차단한다.
- 서버에서도 같은 검증을 다시 수행한다.
- 최대 입력 길이는 1,000자로 제한한다.
- 사용자가 입력한 원문을 화면에 다시 표시할 때는 `innerHTML`을 사용하지 않는다.
- 텍스트 출력은 `textContent`를 사용한다.

## 7. 코딩 스타일 규칙

### 7.1 코드 작성 원칙

- 복잡한 추상화보다 읽기 쉬운 코드를 우선한다.
- 함수 이름은 역할이 드러나게 작성한다.
- 한 함수는 하나의 역할만 맡는다.
- 비전공자도 읽을 수 있도록 핵심 부분에는 짧은 주석을 단다.
- 불필요한 라이브러리를 추가하지 않는다.

### 7.2 파일명 규칙

| 목적 | 파일명 예시 |
|---|---|
| 메인 화면 | `public/index.html` |
| 스타일 | `public/styles.css` |
| 프론트 동작 | `public/app.js` |
| 분석 API | `api/analyze.js` |
| OpenAI 유틸 | `lib/openai.js` |
| Supabase 유틸 | `lib/supabase.js` |

### 7.3 응답 형식 규칙

서버 API는 항상 JSON만 반환한다.  
HTML 에러 페이지를 반환하지 않는다.

성공 응답:

```json
{
  "ok": true,
  "result": {
    "sentiment": "positive",
    "labelKo": "긍정",
    "confidence": 87,
    "reason": "문장 안에 만족과 기대를 나타내는 표현이 포함되어 있습니다."
  },
  "saved": true
}
```

실패 응답:

```json
{
  "ok": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "분석할 텍스트를 입력해주세요."
  }
}
```

## 8. 오류 처리 규칙

| 상황 | 사용자 메시지 |
|---|---|
| 입력값 없음 | 분석할 텍스트를 입력해주세요. |
| 1,000자 초과 | 입력 글자는 최대 1,000자까지 가능합니다. |
| OpenAI API 실패 | 감성 분석 중 문제가 발생했습니다. 잠시 후 다시 시도해주세요. |
| Supabase 저장 실패 | 분석은 완료되었지만 기록 저장에 실패했습니다. |
| 알 수 없는 오류 | 예상하지 못한 문제가 발생했습니다. 다시 시도해주세요. |

에러가 발생해도 사용자가 무엇을 해야 하는지 알 수 있도록 짧고 구체적으로 표시한다.

## 9. UI 구현 규칙

UI는 첨부 이미지의 숲 배경 분위기를 참고한다.

핵심 방향:

- 어두운 숲 이미지 또는 숲 느낌의 그라데이션 배경
- 중앙 정렬된 우아한 흰색 타이틀
- 차분한 감성 서비스 느낌
- 과도한 애니메이션 금지
- 분석 입력 영역은 반투명 카드 형태
- 결과 영역은 명확하게 구분
- 결과 배지는 감성별 지정 색상으로 표시
  - 긍정: `#3D8B5A`
  - 부정: `#A14A3B`
  - 중립: `#6E7781`

대표 카피:

```text
Forest of Words, Reading the Emotion
문장 속 감정의 결을 조용히 분석합니다.
```

## 10. 완료 기준

다음 조건을 모두 만족해야 완료로 판단한다.

- 사용자가 텍스트를 입력하고 분석 버튼을 누를 수 있다.
- 빈 입력값은 서버 요청 없이 오류 메시지를 보여준다.
- 1,000자 초과 입력은 서버 요청 없이 오류 메시지를 보여준다.
- 정상 입력은 `/api/analyze`로 POST 요청을 보낸다.
- 서버는 OpenAI API를 호출한다.
- 서버는 긍정 / 부정 / 중립 중 하나만 반환한다.
- 신뢰도는 0~100 사이 숫자로 반환한다.
- 분석 이유는 한국어 문장으로 반환한다.
- 결과가 화면에 표시된다.
- 긍정 / 부정 / 중립 결과가 지정된 색상 배지와 텍스트 라벨로 함께 표시된다.
- 분석 성공 또는 실패 로그가 Supabase에 저장된다.
- OpenAI API 키와 Supabase Service Role Key가 프론트엔드 번들에 노출되지 않는다.
- Vercel 배포 후 실제 URL에서 동작한다.

## 11. 에이전트 작업 방식

작업할 때 다음 순서로 진행한다.

1. 문서 전체를 읽는다.
2. 구현 범위와 제외 범위를 확인한다.
3. 먼저 폴더 구조를 만든다.
4. 환경 변수 예시 파일을 만든다.
5. Supabase SQL을 문서 기준으로 작성한다.
6. 백엔드 API를 먼저 구현한다.
7. 프론트엔드를 연결한다.
8. UI를 문서 기준으로 다듬는다.
9. 테스트 체크리스트를 기준으로 검증한다.
10. 완료 후 변경 파일과 테스트 결과를 요약한다.
