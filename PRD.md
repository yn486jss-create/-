# PRD.md

# 감성 분석 서비스 제품 요구사항 문서

## 1. 제품명

```text
Sentiment Forest / 감정의 숲
```

---

## 2. 제품 개요

`Sentiment Forest`는 사용자가 입력한 문장을 OpenAI API로 분석하여 감정 상태를 `긍정`, `부정`, `중립` 중 하나로 분류하는 웹 서비스이다.  
분석 결과는 신뢰도 백분율과 짧은 분석 이유로 함께 제공한다.

이 서비스는 학습용 프로젝트이면서 실제 배포까지 가능한 작은 MVP로 만든다.

---

## 3. 문제 정의

사용자는 짧은 문장이나 후기를 읽고 그 문장이 긍정적인지, 부정적인지, 중립적인지 빠르게 파악하고 싶다.  
단순 키워드 검색 방식은 문맥을 놓칠 수 있으므로 OpenAI API를 활용해 문장의 전체적인 감정 흐름을 분석한다.

---

## 4. 대상 사용자

### 4.1 1차 사용자

- 데이터 분석이나 AI 서비스를 처음 배우는 학습자
- HTML, CSS, JavaScript 기반 프로젝트를 만들고 싶은 비전공자
- OpenAI API, Node.js, Supabase, Vercel을 한 번에 실습하고 싶은 사용자

### 4.2 2차 사용자

- 리뷰, 설문 응답, 짧은 의견 문장의 감성 분류를 테스트해보고 싶은 사용자

---

## 5. 사용자 시나리오

1. 사용자가 웹사이트에 접속한다.
2. 숲 배경의 차분한 첫 화면을 본다.
3. 텍스트 입력창에 문장을 입력한다.
4. `감성 분석하기` 버튼을 누른다.
5. 로딩 메시지를 본다.
6. 분석이 완료되면 결과를 확인한다.
7. 결과에는 다음 정보가 표시된다.
   - 긍정 / 부정 / 중립
   - 신뢰도 백분율
   - 분석 이유
8. 입력값이나 서버 문제가 있으면 오류 메시지를 확인한다.

---

## 6. 핵심 기능 요구사항

| ID | 기능 | 설명 | 우선순위 |
|---|---|---|---|
| FR-001 | 텍스트 입력 | 사용자가 분석할 문장을 입력한다. | Must |
| FR-002 | 입력값 검증 | 빈 입력과 1,000자 초과 입력을 차단한다. | Must |
| FR-003 | 분석 요청 | 프론트엔드가 백엔드 API로 분석 요청을 보낸다. | Must |
| FR-004 | OpenAI 분석 | 백엔드가 OpenAI API로 감성 분석을 요청한다. | Must |
| FR-005 | 결과 표시 | 긍정 / 부정 / 중립 결과를 화면에 표시한다. | Must |
| FR-006 | 신뢰도 표시 | 0~100 사이 백분율을 표시한다. | Must |
| FR-007 | 분석 이유 표시 | 결과 판단의 간단한 이유를 한국어로 표시한다. | Must |
| FR-008 | 오류 메시지 표시 | 사용자 입력 오류와 서버 오류를 구분해 표시한다. | Must |
| FR-009 | DB 저장 | 분석 요청과 결과를 Supabase에 저장한다. | Must |
| FR-010 | 배포 | Vercel에서 동작해야 한다. | Must |

## 7. 제외 범위

이번 버전에서는 아래 기능을 만들지 않는다.

| 제외 기능 | 제외 이유 |
|---|---|
| 로그인 | MVP 범위를 단순하게 유지하기 위해 제외 |
| 분석 기록 조회 | DB 저장 검증만 하고 UI 조회는 다음 버전으로 보류 |
| 관리자 페이지 | 초기 학습 프로젝트 범위를 초과 |
| 파일 업로드 | 텍스트 입력 중심 서비스로 제한 |
| 이미지 분석 | 기술 범위가 달라짐 |
| 음성 분석 | 기술 범위가 달라짐 |
| 결제 | 프로젝트 목적과 무관 |
| 복잡한 통계 차트 | 초기 구현 부담 증가 |

## 8. 비기능 요구사항

### 8.1 사용성

- 첫 화면에서 사용자가 바로 무엇을 해야 하는지 알 수 있어야 한다.
- 버튼 문구는 명확해야 한다.
- 오류 메시지는 짧고 이해하기 쉬워야 한다.
- 비전공자도 코드 흐름을 따라갈 수 있어야 한다.

### 8.2 성능

- 일반적인 짧은 문장 기준 10초 이내에 결과를 표시하는 것을 목표로 한다.
- OpenAI 응답 지연 시 로딩 상태를 계속 보여준다.
- 입력 길이는 1,000자로 제한한다.

### 8.3 보안

- OpenAI API Key는 브라우저에 노출되면 안 된다.
- Supabase Service Role Key는 브라우저에 노출되면 안 된다.
- 모든 외부 API 호출은 Node.js 서버 API에서 수행한다.
- 프론트엔드는 `/api/analyze`만 호출한다.

### 8.4 접근성

- 입력창에는 label을 연결한다.
- 버튼은 키보드로 선택 가능해야 한다.
- 결과와 오류 메시지는 시각적으로 구분되어야 한다.
- 배경 이미지 위 텍스트 대비를 확보한다.

Hero 문구:

- 텍스트 입력창

- 감성 분석 버튼
- 오류 메시지 영역

### 9.3 결과 카드

포함 요소:

- 감성 결과 라벨
- 신뢰도 백분율

- 분석 이유

## 10. API 요구사항

### 10.1 엔드포인트

```text
POST /api/analyze
```

### 10.2 요청 Body

```json
{
  "text": "오늘은 날씨가 좋아서 기분이 좋아요."
}
```

### 10.3 성공 응답

```json
{
  "ok": true,
  "result": {
    "sentiment": "positive",
    "labelKo": "긍정",
    "confidence": 92,
    "reason": "날씨가 좋고 기분이 좋다는 표현이 포함되어 있어 긍정으로 판단됩니다."
  },
  "saved": true
}
```

### 10.4 실패 응답

```json
{
  "ok": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "분석할 텍스트를 입력해주세요."
  }
}
```

---

## 11. 데이터 저장 요구사항

Supabase에는 분석 요청과 결과를 저장한다.

저장 항목:

- 입력 텍스트
- 분석 결과
- 신뢰도
- 분석 이유
- 요청 성공 여부
- 에러 코드
- 에러 메시지
- 사용한 OpenAI 모델명
- 생성 시각

상세 스키마는 `docs/05_SUPABASE_DB_SPEC.md`를 따른다.

---

## 12. 성공 지표

| 지표 | 목표 |
|---|---|
| 정상 분석 성공 | 사용자가 입력한 텍스트에 대해 결과 표시 |
| 오류 처리 | 빈 입력, 긴 입력, 서버 오류를 구분 표시 |
| 보안 | API Key가 프론트엔드에 노출되지 않음 |
| DB 저장 | 성공 / 실패 로그가 Supabase에 저장됨 |
| 배포 | Vercel 배포 URL에서 동작 |

---

## 13. 완료 기준

프로젝트 완료는 다음을 모두 만족할 때로 한다.

- `index.html`에서 입력과 결과 확인이 가능하다.
- `/api/analyze`가 JSON 요청과 JSON 응답을 처리한다.
- OpenAI API 응답이 항상 정해진 감성 값 중 하나로 정규화된다.
- Supabase 저장이 성공하거나, 저장 실패 시에도 사용자에게 분석 결과는 보여준다.
- Vercel 환경 변수 설정 후 배포된다.
- `docs/06_DEPLOY_TEST_CHECKLIST.md`의 테스트 항목을 모두 통과한다.
